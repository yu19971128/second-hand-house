# 二手房系统 — 新功能需求文档

## 一、现有功能盘点

| 模块 | 已有功能 |
|------|---------|
| 认证 | 注册、登录、获取当前用户 |
| 房源 | 列表/搜索/筛选、详情、发布、编辑、下架、收藏/取消收藏 |
| 我的 | 我的发布、我的收藏 |
| 管理后台 | 数据统计、房源管理、用户管理 |

---

## 二、可开发的新功能（按难度分级）

---

### Level 1 — 入门级（1-2天）

#### 功能1：用户注册页面

> 当前只有登录页，注册需要直接调接口

**涉及文件：**
- 新建 `mobile/src/pages/Register.jsx`
- `mobile/src/App.jsx` 添加路由 `/register`

**接口：** 后端已有 `POST /api/auth/register`，只需开发前端

**页面字段：**
- 用户名、密码、确认密码、手机号
- 角色选择：买家 / 中介

---

#### 功能2：修改个人信息

> 当前「我的」页面无法修改资料

**涉及文件：**
- `mobile/src/pages/My.jsx` 添加编辑入口
- 新建 `mobile/src/pages/EditProfile.jsx`
- 后端 `server/routes/auth.js` 新增接口

**新增接口：**

```
PUT /api/auth/profile   修改用户名、手机号
PUT /api/auth/password  修改密码（需验证旧密码）
```

**数据库：** 无需改动，`users` 表已有相关字段

---

#### 功能3：房源编辑页面

> 后端已有 `PUT /api/houses/:id`，前端无编辑入口

**涉及文件：**
- 新建 `mobile/src/pages/EditHouse.jsx`（复用 Publish.jsx 的表单）
- `My.jsx` 我的发布列表添加「编辑」按钮

---

### Level 2 — 进阶级（3-5天）

#### 功能4：预约看房

> 买家对感兴趣的房源提交看房预约，中介确认

**数据库新增表：**

```sql
CREATE TABLE appointments (
  id INT PRIMARY KEY AUTO_INCREMENT,
  house_id INT NOT NULL,
  buyer_id INT NOT NULL,
  agent_id INT NOT NULL,
  appoint_time DATETIME NOT NULL,
  status ENUM('pending','confirmed','cancelled','completed') DEFAULT 'pending',
  remark VARCHAR(200) DEFAULT NULL,
  created_at TIMESTAMP DEFAULT CURRENT_TIMESTAMP,
  FOREIGN KEY (house_id) REFERENCES houses(id),
  FOREIGN KEY (buyer_id) REFERENCES users(id)
);
```

**后端新增接口（新建 `routes/appointments.js`）：**

```
POST   /api/appointments          买家提交预约
GET    /api/appointments/my       我的预约列表（买家）
GET    /api/appointments/received 收到的预约（中介）
PUT    /api/appointments/:id      确认/取消预约
```

**前端新增：**
- 房源详情页添加「预约看房」按钮
- 「我的」页面新增「我的预约」Tab

---

#### 功能5：房源留言/评论

> 用户可在房源详情页留言咨询

**数据库新增表：**

```sql
CREATE TABLE comments (
  id INT PRIMARY KEY AUTO_INCREMENT,
  house_id INT NOT NULL,
  user_id INT NOT NULL,
  content VARCHAR(500) NOT NULL,
  created_at TIMESTAMP DEFAULT CURRENT_TIMESTAMP,
  FOREIGN KEY (house_id) REFERENCES houses(id),
  FOREIGN KEY (user_id) REFERENCES users(id)
);
```

**后端新增接口（新建 `routes/comments.js`）：**

```
POST   /api/houses/:id/comments   发表留言
GET    /api/houses/:id/comments   获取留言列表
DELETE /api/comments/:id          删除自己的留言
```

**前端：** `HouseDetail.jsx` 底部添加留言区

---

#### 功能6：浏览历史

> 记录用户最近看过的房源，方便回顾

**数据库新增表：**

```sql
CREATE TABLE view_history (
  id INT PRIMARY KEY AUTO_INCREMENT,
  user_id INT NOT NULL,
  house_id INT NOT NULL,
  viewed_at TIMESTAMP DEFAULT CURRENT_TIMESTAMP ON UPDATE CURRENT_TIMESTAMP,
  UNIQUE KEY unique_view (user_id, house_id),
  FOREIGN KEY (user_id) REFERENCES users(id),
  FOREIGN KEY (house_id) REFERENCES houses(id)
);
```

**后端：** 在 `GET /api/houses/:id` 中，登录用户访问时写入浏览记录

**新增接口：**

```
GET    /api/houses/user/history   获取我的浏览历史
DELETE /api/houses/user/history   清空浏览历史
```

**前端：** 「我的」页面新增「最近浏览」Tab

---

### Level 3 — 挑战级（1周以上）

#### 功能7：站内消息系统

> 买家与中介之间可发送消息

**数据库新增表：**

```sql
CREATE TABLE messages (
  id INT PRIMARY KEY AUTO_INCREMENT,
  from_user_id INT NOT NULL,
  to_user_id INT NOT NULL,
  house_id INT DEFAULT NULL,
  content VARCHAR(1000) NOT NULL,
  is_read TINYINT(1) DEFAULT 0,
  created_at TIMESTAMP DEFAULT CURRENT_TIMESTAMP,
  FOREIGN KEY (from_user_id) REFERENCES users(id),
  FOREIGN KEY (to_user_id) REFERENCES users(id)
);
```

**后端新增接口（新建 `routes/messages.js`）：**

```
POST /api/messages               发送消息
GET  /api/messages/conversations  会话列表
GET  /api/messages/:userId        与某用户的聊天记录
PUT  /api/messages/read/:userId   标记已读
GET  /api/messages/unread-count   未读消息数
```

**前端新增：** 消息列表页、聊天详情页、房源详情添加「联系中介」按钮

---

#### 功能8：房源对比

> 同时对比 2-3 套房源的各项参数

**实现方案（纯前端，无需后端）：**
- 房源列表/详情添加「加入对比」按钮
- 对比数据存 `localStorage`
- 新增对比页面，用表格并排展示价格、面积、户型、地区等

## 三、开发优先级建议

| 优先级 | 功能 | 理由 |
|--------|------|------|
| P0 | 用户注册页面 | 基础功能缺失，新用户无法自助注册 |
| P0 | 房源编辑页面 | 后端已完成，前端缺口 |
| P1 | 修改个人信息 | 用户体验必备 |
| P1 | 房源留言/评论 | 增加买卖互动 |
| P2 | 预约看房 | 核心业务闭环 |
| P2 | 浏览历史 | 提升用户粘性 |
| P3 | 消息系统 | 复杂度高，可最后做 |
| P3 | 房源对比 | 锦上添花 |
